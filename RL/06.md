6강은 **Value Function Approximation**이야.
한 줄로 말하면:

**state/action이 너무 많아서 $V(s)$나 $Q(s,a)$를 table로 저장할 수 없으니, 함수 $\hat v(s;w)$ 또는 $\hat q(s,a;w)$로 근사해서 학습하자**는 내용이야. 슬라이드도 큰 문제에서는 state/action이 너무 많아 table 저장과 개별 학습이 느리다고 설명해. 

---

# 1. 왜 Function Approximation이 필요해?

지금까지는 value function을 table로 저장했어.

$$
V(s)
$$

또는

$$
Q(s,a)
$$

처럼 state마다, state-action마다 값을 하나씩 저장했지.

근데 문제는 실제 RL에서는 state가 엄청 많거나 연속적일 수 있다는 거야.

예를 들어 로봇의 위치, 속도, 관절 각도 같은 건 연속값이니까 가능한 state가 사실상 무한개야. 그러면 모든 state에 대해 table을 만들 수 없어.

그래서 value function을 이런 식으로 바꿔.

$$
\hat v(s;w) \approx v_\pi(s)
$$

또는

$$
\hat q(s,a;w) \approx q_\pi(s,a)
$$

여기서 $w$는 함수의 parameter야.
즉, table의 각 칸을 직접 업데이트하는 게 아니라 **함수의 parameter $w$를 업데이트**하는 방식으로 바뀌는 거야.

---

# 2. 핵심 변화: table update → parameter update

이전에는 이렇게 했어.

$$
V(s) \leftarrow V(s)+\alpha(\text{target}-V(s))
$$

이건 특정 state $s$의 table entry 하나만 바꾸는 거야.

6강에서는 이렇게 바뀌어.

$$
w \leftarrow w + \Delta w
$$

즉, $V(s)$ 하나를 직접 바꾸는 게 아니라, value function을 만들어내는 함수 자체의 parameter를 바꿔.

이게 중요한 이유는 **generalization** 때문이야.

예를 들어 어떤 state $s$에서 학습했는데, parameter $w$가 바뀌면 비슷한 다른 state의 value도 같이 바뀔 수 있어. 그래서 본 적 없는 state에도 어느 정도 일반화가 가능해.

---

# 3. 어떤 함수 approximator를 쓸 수 있나?

슬라이드에서는 여러 예시가 나와.

* Linear combination of features
* Neural network
* Decision tree
* Nearest neighbor
* Fourier / wavelet bases

다만 강의에서는 주로 **differentiable function approximator**, 특히 linear function approximation과 neural network 같은 걸 생각해. 왜냐하면 gradient descent로 $w$를 업데이트해야 하기 때문이야. 

---

# 4. Gradient Descent 기본 아이디어

우리는 approximate value function이 true value function과 비슷해지길 원해.

그래서 loss를 이렇게 잡아.

$$
J(w)=\mathbb{E}*\pi\left[(v*\pi(s)-\hat v(s;w))^2\right]
$$

즉,

> 진짜 value $v_\pi(s)$와 예측 value $\hat v(s;w)$의 차이를 줄이자

는 거야.

Gradient descent는 parameter $w$를 loss가 줄어드는 방향으로 조금씩 움직여.

업데이트 형태는 대략:

$$
\Delta w
========

\alpha
\left(v_\pi(s)-\hat v(s;w)\right)
\nabla_w \hat v(s;w)
$$

의미는:

> prediction error가 크면 많이 고치고, 그 state에서 value 예측에 영향을 준 parameter 방향으로 업데이트한다.

---

# 5. 그런데 RL에는 true value가 없다

여기서 중요한 문제가 있어.

위 식에는 $v_\pi(s)$가 들어가는데, RL에서는 true value를 몰라.

우리가 아는 건 episode에서 얻은 reward뿐이야.

그래서 실제로는 $v_\pi(s)$ 대신 **target**을 넣어.

6강에서는 4강에서 배운 MC, TD, TD($\lambda$) target을 그대로 가져와서 $w$ 업데이트에 쓴다. 

---

# 6. MC with Function Approximation

MC에서는 target이 return $G_t$야.

기존 table 방식:

$$
V(S_t)
\leftarrow
V(S_t)+\alpha(G_t-V(S_t))
$$

Function approximation 방식:

$$
\Delta w
========

\alpha
\left(G_t-\hat v(S_t;w)\right)
\nabla_w \hat v(S_t;w)
$$

즉, 현재 함수가 예측한 값 $\hat v(S_t;w)$와 실제 episode 끝까지 본 return $G_t$의 차이만큼 $w$를 조정해.

MC target $G_t$는 true value의 noisy sample이지만 unbiased sample로 볼 수 있어서, supervised learning처럼 다룰 수 있다고 설명돼. 

---

# 7. TD(0) with Function Approximation

TD에서는 target이 이거였지.

$$
R_{t+1}+\gamma V(S_{t+1})
$$

Function approximation에서는 $V$ 대신 $\hat v$를 써.

$$
\text{TD target}
================

R_{t+1}+\gamma \hat v(S_{t+1};w)
$$

업데이트는:

$$
\Delta w
========

\alpha
\left(
R_{t+1}
+
\gamma \hat v(S_{t+1};w)
------------------------

\hat v(S_t;w)
\right)
\nabla_w \hat v(S_t;w)
$$

여기서 괄호 안이 TD error야.

$$
\delta_t
========

R_{t+1}
+
\gamma \hat v(S_{t+1};w)
------------------------

\hat v(S_t;w)
$$

그래서 간단히 쓰면:

$$
\Delta w
========

\alpha \delta_t \nabla_w \hat v(S_t;w)
$$

이게 6강에서 중요한 식이야.

---

# 8. Linear Value Function Approximation

가장 단순한 function approximator가 linear model이야.

state $s$를 feature vector로 바꿔.

$$
x(s)=
\begin{bmatrix}
x_1(s)\
x_2(s)\
\vdots\
x_n(s)
\end{bmatrix}
$$

그리고 value를 feature들의 weighted sum으로 표현해.

$$
\hat v(s;w)=x(s)^T w
$$

즉,

$$
\hat v(s;w)=x_1(s)w_1+x_2(s)w_2+\cdots+x_n(s)w_n
$$

이 경우 gradient가 아주 단순해.

$$
\nabla_w \hat v(s;w)=x(s)
$$

그래서 업데이트가:

$$
\Delta w
========

\alpha
\left(\text{target}-x(S_t)^T w\right)x(S_t)
$$

가 돼.

이걸 말로 하면:

> 업데이트 = learning rate × prediction error × feature vector

이야. 슬라이드에서도 linear approximation의 업데이트를 “step-size × prediction error × feature value”로 정리해. 

---

# 9. Linear MC / Linear TD 식

Linear MC:

$$
\Delta w
========

\alpha
\left(G_t-x(S_t)^T w\right)x(S_t)
$$

Linear TD(0):

$$
\Delta w
========

\alpha
\left(
R_{t+1}
+
\gamma x(S_{t+1})^T w
---------------------

x(S_t)^T w
\right)
x(S_t)
$$

즉, MC냐 TD냐의 차이는 target만 달라.

| 방법            | target                             |
| ------------- | ---------------------------------- |
| MC            | $G_t$                              |
| TD(0)         | $R_{t+1}+\gamma \hat v(S_{t+1};w)$ |
| TD($\lambda$) | $G_t^\lambda$                      |

---

# 10. Control with Function Approximation

Prediction에서는 $\hat v(s;w)$를 배웠지.

Control에서는 action을 골라야 하니까 $\hat q(s,a;w)$를 배워야 해.

$$
\hat q(s,a;w)\approx q_\pi(s,a)
$$

그리고 policy improvement는 이전처럼 $\epsilon$-greedy로 하면 돼.

$$
\pi(s)=\epsilon\text{-greedy}(\hat q(s,\cdot;w))
$$

즉,

1. 현재 $w$로 $\hat q(s,a;w)$를 계산한다.
2. 가장 큰 action을 주로 고른다.
3. 경험을 모은다.
4. MC/SARSA/Q-learning target으로 $w$를 업데이트한다.

슬라이드에서도 control은 $q_\pi$를 estimate하고, approximate policy evaluation by $\hat q(\cdot,\cdot;w)$를 한 뒤, $\epsilon$-greedy policy improvement를 한다고 설명해. 

---

# 11. Action-Value Function Approximation

Action-value도 똑같이 MSE를 줄이는 식으로 생각해.

$$
J(w) = \mathbb{E}_\pi
\left[
(q*\pi(s,a)-\hat q(s,a;w))^2
\right]
$$

업데이트는:

$$
\Delta w =
\alpha
\left(
q_\pi(s,a)-\hat q(s,a;w)
\right)
\nabla_w \hat q(s,a;w)
$$

하지만 $q_\pi(s,a)$도 모르니까 실제로는 target을 넣어.

예를 들어 SARSA target:

$$
R_{t+1}+\gamma \hat q(S_{t+1},A_{t+1};w)
$$

Q-learning target:

$$
R_{t+1}+\gamma \max_{a'}\hat q(S_{t+1},a';w)
$$

그래서 Q-learning with function approximation은 대략:

$$
\Delta w =
\alpha
\left[
R_{t+1}
+
\gamma \max_{a'}\hat q(S_{t+1},a';w)-
\hat q(S_t,A_t;w)
\right]
\nabla_w \hat q(S_t,A_t;w)
$$

가 돼.

---

# 12. Batch Methods

앞부분은 대부분 **incremental method**야.

즉, sample 하나 또는 step 하나가 들어올 때마다 조금씩 업데이트하는 방식이야.

하지만 incremental SGD는 간단한 대신 sample efficient하지 않을 수 있어. 즉, 수렴까지 오래 걸릴 수 있어. 그래서 batch method는 지금까지 모은 experience 전체, 즉 training data를 가지고 더 잘 맞는 value function을 찾으려고 해. 

대표 아이디어는:

> 경험을 buffer에 저장해두고, 그 데이터 전체에 대해 value function이 잘 맞도록 학습한다.

이때 같은 데이터를 여러 번 재사용할 수 있어. 이게 나중에 experience replay 같은 아이디어와도 연결돼.

---

# 13. Linear Least Squares Prediction

linear value approximation에서는

$$
\hat v(s;w)=x(s)^T w
$$

였지.

이 경우에는 MSE를 최소화하는 $w$를 gradient descent로 천천히 찾는 대신, least squares로 직접 풀 수도 있어.

즉, 데이터가

$$
(S_1, y_1), (S_2, y_2), \cdots, (S_n, y_n)
$$

처럼 있고 target $y_i$가 있다면,

$$
x(S_i)^T w \approx y_i
$$

가 되도록 $w$를 찾는 거야.

MC라면 target $y_i=G_i$가 되고, TD라면 TD target을 쓰게 돼.

슬라이드에서도 experience replay는 least squares solution을 찾지만 여러 iteration이 필요할 수 있고, linear value approximation에서는 least squares solution을 직접 풀 수 있다고 설명해. 

---

# 14. 6강 전체를 하나로 연결하면

4강, 5강에서는 table 기반으로 배웠어.

$$
V(s),\quad Q(s,a)
$$

6강에서는 이걸 함수로 바꿔.

$$
\hat v(s;w),\quad \hat q(s,a;w)
$$

그리고 업데이트도 table entry가 아니라 parameter $w$를 바꾸는 형태로 바뀌어.

공통 형태는 거의 항상 이거야.

$$
\Delta w
========

\alpha
\left(
\text{target}
-------------

\text{prediction}
\right)
\nabla_w \text{prediction}
$$

여기서 target만 알고리즘마다 달라.

| 알고리즘               | prediction          | target                                        |
| ------------------ | ------------------- | --------------------------------------------- |
| MC prediction      | $\hat v(S_t;w)$     | $G_t$                                         |
| TD prediction      | $\hat v(S_t;w)$     | $R_{t+1}+\gamma\hat v(S_{t+1};w)$             |
| SARSA control      | $\hat q(S_t,A_t;w)$ | $R_{t+1}+\gamma\hat q(S_{t+1},A_{t+1};w)$     |
| Q-learning control | $\hat q(S_t,A_t;w)$ | $R_{t+1}+\gamma\max_{a'}\hat q(S_{t+1},a';w)$ |

---

# 시험용 핵심 정리

6강에서 제일 중요한 건 이거야.

**Table-based RL**

$$
V(s) \leftarrow V(s)+\alpha(\text{target}-V(s))
$$

**Function approximation RL**

$$
w \leftarrow w+\alpha(\text{target}-\hat v(s;w))\nabla_w\hat v(s;w)
$$

Linear case에서는:

$$
\hat v(s;w)=x(s)^T w
$$

$$
\Delta w=\alpha(\text{target}-x(s)^T w)x(s)
$$

즉, 6강은 **“state마다 값을 저장하지 말고, feature와 parameter로 value function을 근사하자”**가 핵심이야.
